# 프로세스 가이드

이 저장소는 두 가지 모드를 지원한다.

## 모드 선택

- **신규 프로젝트**: 아래 1~7단계를 순서대로 진행한다.
- **기존 코드 리팩토링**: `templates/refactor-start.md` 한 개로 시작한 뒤, 곧바로 `.claude/skills/plan-and-build/SKILL.md` 흐름(논의 → task 생성 → `_runner/run-phases.py` 실행)으로 넘어간다. 1~6단계는 건너뛴다.

## 전제조건

- 이 저장소를 클론하여 프로젝트 디렉토리로 사용할 것
- Claude Code 환경이 세팅되어 있을 것

## 진행 순서 (신규 프로젝트)

### 1단계: 초기 기획 — `templates/1-초기기획.md`

목적: 무엇을 만들 것인지 정의하고, 기술적 구현 가능성을 검증한다.
AI 역할: 공동창업자 / 기술검증자
할 일: 템플릿의 `{중괄호}` 부분을 채워서 AI에게 전달한다.
핵심: AI가 구현 가능성을 점검하고, 모호한 부분을 전부 질문하게 한다.
산출물: 구현 가능성 확인 + 기술적 결정사항 초안

### 2단계: 사용 흐름 설계 — `templates/2-사용흐름.md`

목적: 사용자가 앱/서비스를 어떻게 쓰는지, 모든 경로를 명확히 한다.
AI 역할: 시니어 UX 리서처
할 일: 템플릿을 그대로 복붙한다. 프로젝트별 수정 거의 없음.
핵심: AI가 흐름을 생생히 재현할 수 있을 정도로 명확해질 때까지 질문하게 한다.
산출물: 확정된 사용자 흐름

### 3단계: 화면 구성 설계 — `templates/3-화면구성.md`

목적: 각 화면에 어떤 정보가 표시되고 어떤 기능이 있는지 확정한다.
AI 역할: 시니어 UX 리서처 (2단계와 동일)
할 일: 템플릿을 그대로 복붙한다. 프로젝트별 수정 거의 없음.
핵심: 화면 단위로 정보/기능을 점검하고, 빠진 부분을 잡아낸다.
산출물: 확정된 화면 구성

### 4단계: 아키텍처 + 데이터 설계 — `templates/4-아키텍쳐설계.md`

목적: 코드 구조, 데이터 스키마, 기술 스택을 확정한다.
AI 역할: 시니어 CTO
할 일: 템플릿을 그대로 복붙한다. 프로젝트별 수정 거의 없음.
핵심: 최대한 간결하고 빠르게 구현할 수 있는 설계를 하되, 논의할 점을 전부 잡아낸다.
산출물: 확정된 아키텍처 + 데이터 스키마

### 5단계: 기술 결정사항 정리 — `templates/5-기술결정정리.md`

목적: 지금까지 내린 모든 기술적 결정을 한 곳에 정리한다.
할 일: 템플릿을 그대로 복붙한다.
산출물: 기술 결정사항 요약

### 6단계: 문서 생성 — `templates/6-문서화.md`

목적: AI 에이전트가 읽을 설계 문서를 `spec/`에 생성한다.
할 일: 템플릿을 복붙한다. 필요시 추가 문서 종류를 지정한다.
기본 문서 5종: prd, flow, data-schema, code-architecture, adr
작성 원칙: AI 에이전트를 위한 문서. 간결하되 의사결정 의도는 보존.
산출물: `spec/` 디렉토리에 문서 파일들

### 7단계: 구현 시작 — `templates/7-구현시작.md`

목적: plan-and-build skill을 사용하여 구현을 시작한다.
할 일: 템플릿을 그대로 복붙한다.
이후: SKILL.md의 3~6번 흐름을 따른다.

## 참고사항

- 1~5단계는 하나의 대화 세션에서 연속으로 진행한다.
- 각 단계에서 AI가 질문하면 답하고, 충분히 명확해졌다고 판단되면 다음 단계 템플릿을 붙여넣는다.
- 간단한 프로젝트는 각 단계의 대화가 짧게 끝날 뿐, 단계를 건너뛰지는 않는다.
- 2~7번 템플릿은 대부분 그대로 복붙 가능. 1번만 프로젝트마다 새로 작성한다.
