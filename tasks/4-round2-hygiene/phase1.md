# Phase 1: repo-hygiene

## 사전 준비

먼저 아래 문서를 읽어 설계 의도를 파악하라:

- `spec/refactor-round1.md` (§3 out-of-scope 목록과 §8 "다음 라운드로 미루는 항목"이 이번 라운드의 출발점이다)
- `CLAUDE.md` (프로젝트 규칙)
- `.gitignore` (기존 패턴 확인, 제거 금지)

그리고 아래 명령으로 현재 저장소 위생 상태를 직접 파악하라. source-first다:

```bash
git ls-files | grep -E '^(debug_|result_utf8|inspection_result|null_analysis_result|moef_dump|agencies_history|agency_stats|sdf\.md|image\.png)'
git ls-files | grep -E '^web/check_'
git ls-files | grep -E '^scripts/debug/'
git ls-files | grep -E '^docs/.*\.(resolved|metadata\.json|webp|png)'
ls docs/
ls scripts/
```

이전 phase 산출물: 없음 (이번 task의 첫 phase).

문서보다 코드가 우선이다. 의문점은 작업 중 기록하라.

## 작업 내용

이 phase는 **저장소 위생 정리만** 다룬다. 코드 로직은 건드리지 않는다. 삭제·이동 + `.gitignore` scoped 패턴 추가 + `.env.example` 2개 생성이 전부.

### 1. 루트 dump 파일 삭제 (`git rm`)

아래 파일들이 tracked라면 `git rm`으로 삭제하라. 존재하지 않으면 건너뛴다.

- `debug_ifrs.md`
- `debug_ifrs_result.md`
- `debug_run_v2.py`
- `result_utf8.txt`
- `inspection_result.txt`
- `null_analysis_result.txt`
- `moef_dump.txt`
- `agencies_history.txt`
- `agency_stats.json`

아래 워킹트리 untracked 파일은 **`git rm`이 아니라 일반 `rm`으로 삭제**하라 (tracked 상태가 아니므로 `git rm`은 실패한다):

- `sdf.md`
- 루트 `image.png`

**`tasks/3-auth-hardening/` 는 이번 라운드 scope 밖이다.** 해당 디렉토리 안의 `image.png`를 포함해 그 디렉토리의 어떤 파일도 건드리지 마라. 과거 완료된 task의 워크스페이스다.

### 2. `web/` orphan 스크립트 삭제 (`git rm`)

- `web/check_db.js`
- `web/check_db_v2.js`
- `web/check_analysis.js`
- `web/check_article.js`
- `web/check_today.js`

### 3. `scripts/debug/` archive 이동 (`git mv`)

- `scripts/debug/` 디렉토리 전체 → `scripts/archive/debug-round0/`
- 디렉토리는 `git mv scripts/debug scripts/archive/debug-round0` 형태로 한 번에 이동한다. `scripts/archive/` 디렉토리가 없으면 먼저 생성.

### 4. `docs/` 파생물 archive 이동 (`git mv`)

`docs/archive/round0/resolved/`와 `docs/archive/round0/screens/` 두 디렉토리를 만든다.

**`docs/archive/round0/resolved/`로 이동:**

- `docs/*.resolved`
- `docs/*.resolved.[0-9]` (0~9까지 존재하는 것 전부)
- `docs/*.metadata.json`

**`docs/archive/round0/screens/`로 이동:**

- `docs/*.webp`
- `docs/*.png` (단, 아래 "유지 대상"은 제외)

**`docs/` 본문으로 유지할 것 (이동·삭제 금지):**

- `docs/ARCHITECTURE.md`
- `docs/MASTER_CONTEXT.md`
- `docs/PRD.md`
- `docs/REQUIREMENTS.md`
- `docs/SCHEMA.md`
- `docs/implementation_plan.md`
- `docs/task.md`

본문 `.md`는 **내용도 위치도 수정 금지**. 이번 phase는 자매 파일(`.resolved*`, `.metadata.json`)과 스크린샷만 archive로 옮긴다.

### 5. `.gitignore` scoped 패턴 추가

기존 라인은 **절대 삭제하지 말고** 파일 끝에 아래 블록을 append하라. **전역 패턴(`*.txt`, `*.png`, `*.webp`) 금지** — `requirements.txt`나 정식 자산을 오차단할 수 있다.

```
# --- Round 2: scratch/dump re-entry guard (scoped to repo root) ---
/debug_*.md
/debug_*.py
/*_result.txt
/moef_dump.txt
/agencies_history.txt
/agency_stats.json
/sdf.md
/image.png

# --- Round 2: docs snapshot artifacts (auto-generated by prior tooling) ---
docs/*.resolved
docs/*.resolved.*
docs/*.metadata.json

# --- Round 2: current task workspace scratch images (scoped to this task only) ---
/tasks/4-round2-hygiene/**/image*.png
```

### 6. `.env.example` 2개 생성

**루트 `.env.example`** (실제 값 금지, placeholder만):

```
SUPABASE_URL=https://<project>.supabase.co
SUPABASE_ANON_KEY=<anon-jwt>
GEMINI_API_KEY=<gemini-api-key>
TELEGRAM_BOT_TOKEN=<optional>
TELEGRAM_CHAT_ID=<optional>

```

**`web/.env.local.example`**:

```
NEXT_PUBLIC_SUPABASE_URL_V2=https://<project>.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY_V2=<anon-jwt>
SUPABASE_SERVICE_ROLE_KEY=<service-role-jwt>
```

`GEMINI_ANALYZER_MODEL` / `GEMINI_REPORT_MODEL` 같은 env는 Phase 7에서 추가한다. 이번 phase는 여기까지.

## AC 검증 방법

위 AC 커맨드를 순서대로 실행하라. 모두 통과하면 `tasks/4-round2-hygiene/index.json`의 phase 1 status를 `"completed"`로 변경하라.

수정 3회 이상 시도해도 실패하면 status를 `"error"`로 변경하고 `error_message` 필드에 구체적인 실패 내용을 기록하라.

사용자 개입이 필요한 상황(권한 문제 등)이 발생하면 status를 `"blocked"`, `blocked_reason`에 사유를 기록하고 즉시 중단하라.

## 주의사항

- **`src/`, `config/`, `spec/`, `_runner/`, `tests/`, `.github/workflows/` 전부 수정 금지.** 이번 phase는 저장소 위생만.
- **`tasks/` 아래는 이번 task 디렉토리(`tasks/4-round2-hygiene/`)와 `tasks/index.json`만 수정 가능.** 과거 task 디렉토리(`tasks/1-backend-structure/`, `tasks/2-moef-source/`, `tasks/3-auth-hardening/`)의 어떤 파일도 삭제·이동·수정하지 마라. `tasks/3-auth-hardening/image.png`도 scope 밖이다.
- **`docs/` 본문 `.md`의 내용 수정 금지.** 오직 `.resolved*`, `.metadata.json`, 스크린샷만 이동.
- **`scripts/` 루트 파일(`fix_env_encoding.py`, `seed_v2_data.py`, `reanalyze_null_articles.py`, `full_migration_v2.py`, `make_transparent*.py` 등) 건드리지 마라.** scope 밖으로 두기로 결정된 항목이다.
- **`.gitignore` 전역 패턴 추가 금지.** `*.txt`, `*.png`, `*.webp`, `*.json` 같은 확장자 전역 패턴을 쓰면 `requirements.txt`, 정식 자산, `config/agencies.json` 등이 오차단된다. 반드시 루트 앵커(`/`) 또는 디렉토리 prefix 기반 scoped 패턴만 사용.
- **`config/agencies.json` 수정 금지.** 데이터는 Round 2 내내 불변.
- **파일 이동 후 `from src.pipeline import Pipeline` import가 여전히 성공하는지 반드시 확인.** scripts/debug 이동 과정에서 `src/` import를 깨면 안 된다.
- 기존 테스트를 깨뜨리지 마라 (현 시점에는 pytest가 없지만, import smoke는 반드시 유지).
- 이동·삭제는 반드시 `git mv` / `git rm`으로 수행. 일반 `mv` / `rm` 사용 금지 (이력 추적 깨짐).
